# Implementation Plan: Browser Offline Identity Client

**Branch**: `003-browser-offline-client` | **Date**: 2026-06-24 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-browser-offline-client/spec.md`

## Summary

Build a browser-side JavaScript library that verifies W3C DIDs against a locally-stored
trust bundle, with zero network calls during verification. The library registers a Service
Worker to handle background bundle sync, stores bundles in IndexedDB, and verifies Ed25519
signatures and SHA-256 Merkle inclusion proofs using the Web Crypto API (with `@noble/ed25519`
fallback for older browsers). Delivered as ESM + IIFE builds via esbuild.

## Technical Context

**Language/Version**: JavaScript (ES2022); TypeScript type declarations included

**Primary Dependencies**:
- `cbor-x` — CBOR decode (bundle deserialization)
- `idb` — IndexedDB Promise wrapper
- `@noble/ed25519` — Ed25519 fallback for Safari < 17 / Firefox < 129
- `esbuild` — build tooling (dev dependency only)
- `vitest` — unit testing
- `@playwright/test` — browser integration testing (dev dependency)

**Storage**: IndexedDB (`decpki` DB, `bundles` + `meta` object stores)

**Testing**: Vitest + jsdom (unit); Playwright (browser integration)

**Target Platform**: Browser — Chrome 113+, Firefox 129+, Safari 17+ (noble fallback for older)

**Project Type**: Client-side library (ESM + IIFE) + Service Worker script

**Performance Goals**: `verify()` < 500ms for 10k-identity bundle; bundle sync < 5s broadband

**Constraints**: No server-side runtime; no build toolchain required by consumers; < 30KB
gzipped for the main client module, < 15KB for the SW script

**Scale/Scope**: Single-page app integration; up to 10k identities per bundle

    │   ├── crypto.test.js      # Merkle + Ed25519 unit tests

**Structure Decision**: Single directory (`browser/`) containing the JS library, SW, demo,
and tests. Sits alongside the existing Python package (`src/decpki/`) at the repo root.
This keeps both projects visible without introducing a monorepo tooling layer.
