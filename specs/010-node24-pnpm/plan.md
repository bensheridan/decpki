# Implementation Plan: Node 24 + pnpm Migration

**Branch**: `010-node24-pnpm` | **Date**: 2026-06-29 | **Spec**: [spec.md](spec.md)

## Summary

Upgrade the browser toolchain from Node 18 to Node 24 and switch the package manager from npm
to pnpm. Remove the Node 18-era version pins on vitest/happy-dom, add `engines` and
`packageManager` fields to `browser/package.json`, delete `package-lock.json`, commit
`pnpm-lock.yaml`, and update all documentation and scripts that reference `npm install`.

## Technical Context

**Language/Version**: Node.js 24 LTS

**Package Manager**: pnpm (latest stable — install via `corepack enable pnpm`)

**Testing**: vitest (latest, unpinned)

**Target Platform**: macOS / Linux developer workstation

**Constraints**: Zero changes to Python/BFF code. No new runtime dependencies.

**Scale/Scope**: Five files changed, one file deleted, one file created.

## Implementation Notes

### `browser/package.json` changes

Add `engines` and `packageManager` fields; restore devDependencies to latest semver ranges:

```json
{
  "engines": { "node": ">=24.0.0" },
  "packageManager": "pnpm@<version-at-migration-time>",
  "devDependencies": {
    "@vitest/coverage-v8": "^4.1.9",
    "esbuild": "^0.28.1",
    "happy-dom": "^20.10.6",
    "vitest": "^4.1.9"
  }
}
```

### `browser/.npmrc`

```ini
engine-strict=true
```

Causes `npm install` to hard-fail on engine mismatch, steering contributors toward pnpm.

### `scripts/start-demo.sh` addition

```bash
require_cmd pnpm "Run: corepack enable pnpm  (Node 24 includes corepack)"
```

### README change

Replace `cd browser && npm install && cd ..` with `cd browser && pnpm install && cd ..`.
