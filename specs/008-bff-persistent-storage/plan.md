# Implementation Plan: BFF Persistent Storage

**Branch**: `008-bff-persistent-storage` | **Date**: 2026-06-28 | **Spec**: [spec.md](spec.md)

## Summary

Replace the four in-memory dicts in `SessionStore` with a SQLite-backed implementation
(`SqliteSessionStore`) that persists sessions, challenges, and the JTI blocklist across BFF
process restarts. The public interface is unchanged so `main.py` needs only a one-line swap.

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**: `sqlite3` (stdlib) — zero new packages. `python-jose`, `cryptography`,
`fastapi`, `uvicorn` already present.

**Storage**: SQLite file at `$BFF_STORE_PATH` (default `/tmp/decpki-bff.db`). In-memory
(`:memory:`) used for unit tests.

**Testing**: `pytest` (existing); new integration fixture writes to a `tmp_path` SQLite file.

**Target Platform**: macOS / Linux (dev demo server)

**Performance Goals**: Startup restore < 500 ms with 1 000 rows (SC-003).

**Constraints**: Single BFF process; no horizontal scaling. Zero new pip dependencies.

**Scale/Scope**: Demo environment — dozens of sessions at most.

## Implementation Notes

### Schema (bff/session.py)

```sql
CREATE TABLE IF NOT EXISTS challenges (
    challenge_hex TEXT PRIMARY KEY,
    did TEXT NOT NULL,
    credential_id TEXT NOT NULL,
    public_key_hex TEXT NOT NULL,
    algorithm TEXT NOT NULL DEFAULT 'ed25519',
    issued_at INTEGER NOT NULL,
    expires_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS refresh_tokens (
    token_hex TEXT PRIMARY KEY,
    did TEXT NOT NULL,
    issued_at INTEGER NOT NULL,
    expires_at INTEGER NOT NULL,
    last_jti TEXT NOT NULL DEFAULT ''
);
CREATE INDEX IF NOT EXISTS idx_rt_did ON refresh_tokens(did);

CREATE TABLE IF NOT EXISTS jti_blocklist (
    jti TEXT PRIMARY KEY,
    added_at INTEGER NOT NULL,
    original_exp INTEGER NOT NULL
);
```

### SqliteSessionStore skeleton

```python
class SqliteSessionStore:
    def __init__(self, path: str | None = None):
        self._path = path or os.environ.get("BFF_STORE_PATH", "/tmp/decpki-bff.db")
        self._conn = sqlite3.connect(self._path, check_same_thread=False)
        self._conn.execute("PRAGMA journal_mode=WAL")
        self._lock = threading.Lock()
        self._init_schema()
        self._prune_expired()

    def close(self):
        self._conn.close()

    def _prune_expired(self):
        now = int(time.time())
        with self._lock, self._conn:
            self._conn.execute("DELETE FROM challenges WHERE expires_at < ?", (now,))
            self._conn.execute("DELETE FROM refresh_tokens WHERE expires_at < ?", (now,))
            self._conn.execute("DELETE FROM jti_blocklist WHERE original_exp < ?", (now,))
```

### JTI blocklist — storing original_exp

`revoke_session` has access to `last_jti` from the stored refresh token record. The
`original_exp` for the blocklist entry can be approximated as `now + SESSION_LIFETIME_SECONDS`
(conservative upper bound) to avoid changing `revoke_session`'s call signature in `main.py`.

Cross-restart integration test (new file `test_sessions_persist.py`):

```python
def test_refresh_token_survives_restart(tmp_path):
    path = str(tmp_path / "bff.db")
    s1 = SqliteSessionStore(path=path)
    token, _ = s1.issue_refresh_token("did:local:test")
    s1.close()

    s2 = SqliteSessionStore(path=path)
    assert s2.consume_refresh_token(token) is not None
    s2.close()
```
